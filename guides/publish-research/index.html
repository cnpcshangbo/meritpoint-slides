Real scientists don't keep results in a drawer — they publish. This guide takes your Guide 03 project (question + chart + finding) and puts it on a real public website, hosted free by GitHub Pages. You will build the page with the generator in section 3 — no HTML knowledge needed — then publish it two ways: the click-only way (works from zero), or the engineer way with git push. At the end you'll have a link like https://YOURNAME.github.io/my-research/ to send to anyone.

Why GitHub Pages? It hosts websites for free, straight out of a GitHub repo — the same repos you learned in Guides 01/02. Millions of real project sites work exactly this way (including the lab page above — and this very site you're reading).

Quiz — why must chart.png sit in the SAME folder as index.html?

Guide 01, section 2 strikes again: chart.png is a relative path — "look right here, next to me." Move the picture elsewhere and the page shows a broken image. Paths follow you everywhere in computing.

4Publish — Path A: clicks only (works from zero)

Quiz — your repo is called my-research and your username is pixelkid. Where does your site live?

GitHub Pages addresses are always USERNAME.github.io/REPONAME/. The repo page (github.com/…) shows your files; the github.io address shows your website.

5Publish — Path B: the engineer way (git)

Path A works, but engineers publish from the terminal — because they update sites constantly, and clicking through upload forms gets old fast. You already know every move from Guides 01/02; this just chains them:

cd my-research-folder        # the folder with index.html + chart.png
git init                     # save slot (once)
git add .
git commit -m "publish my research"
git remote add origin git@github.com:YOURNAME/my-research.git   # connect the cloud (once)
git push -u origin main      # upload — then enable Pages as in Path A step 3
⚡ Try it now: the Practice Terminal (bottom-right) has your index.html and chart.png waiting. Run the whole publish ritual — Mission: The publish ritual checks you off step by step. Do it twice until it's automatic.
SSH key needed? If git push asks for passwords or gets rejected, that's the keycard from Guide 01 section 5 / Guide 02 section 6: make a key with ssh-keygen and add it to GitHub once, and pushing works forever after.

6Update it, show it off

Here is the beautiful part — your site rebuilds itself every time you push:

# found a better chart? new finding? edit your files, then:
git add .
git commit -m "add second chart"
git push        # ~1 minute later the website shows the new version
